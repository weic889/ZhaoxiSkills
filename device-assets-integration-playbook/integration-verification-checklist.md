# 资产类应用集成验证清单

这份清单用于配合 `device-assets-integration-playbook` 使用，适合在每次联调、提测、上线验证时逐项勾选。

## 1. 接入建模

- [ ] 明确应用运行方式：壳页直出 / 跨域 iframe / 微前端 SDK
- [ ] 明确当前园区真值来源
- [ ] 明确左树接口是否来自平台统一位置服务
- [ ] 明确列表、新增、编辑共用的位置字段模型

## 2. 前端上下文

- [ ] `x-organization-id` 不是直接优先取 token
- [ ] 跨域 iframe 场景已接入宿主 runtime 或 `postMessage`
- [ ] 宿主园区切换后，页面能重新拉树和列表
- [ ] 菜单选中态在二级页面跳转后不丢失

## 3. 后端保存

- [ ] 新增接口会落 `projectId`
- [ ] 新增接口会落 `buildingId`
- [ ] 新增接口会落 `floorId`
- [ ] 新增接口会落 `siteId`
- [ ] 编辑接口不会把园区字段改空或改错

## 4. 后端查询

- [ ] 列表按 `project_id` 或等价组织字段过滤
- [ ] 历史数据兼容逻辑有明确边界
- [ ] 位置树相关查询不会绕过园区过滤

## 5. 配置与部署

- [ ] 线上网关路径正确
- [ ] 服务读取的是目标环境配置
- [ ] 数据库连接的是目标环境库
- [ ] 必要表已存在
- [ ] CI / 镜像 / webhook 已完成

## 6. 浏览器线上验证

- [ ] 壳页展示的园区名称正确
- [ ] 树接口请求参数中的园区正确
- [ ] 列表接口请求头中的 `x-organization-id` 正确
- [ ] 列表返回的数据属于当前园区
- [ ] 新增资产后刷新页面仍属于当前园区
- [ ] 切换到另一个园区后旧园区数据不残留

## 7. 故障定位判断

如果壳页园区正确，但请求头园区错误：

- 结论：前端上下文传递有问题

如果请求头园区正确，但返回数据错误：

- 结论：后端保存字段或查询过滤有问题

如果树切了园区，列表没切：

- 结论：页面缓存、依赖更新或组件重挂有问题

如果接口路径 404 / 504：

- 结论：先看网关前缀、部署状态、配置和数据库
