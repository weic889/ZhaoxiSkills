# PCApp 后端

## 概述

这个技能用于处理基座 PC 应用后端微服务的接入、联调和线上排障。

核心原则：

1. 查询和保存都必须落明确的组织范围字段。
2. 网关路径、服务配置、数据库连接要当成一条链来排查。
3. 线上接口返回正确，比本地单测通过更重要。
4. 后端返回给 PC 壳页的字段语义、展示文案和权限结果必须稳定，不能逼前端在业务仓长期兜底。

## 何时使用

- 新建或接入一个给 PC 壳应用使用的后端服务
- 列表、新增、编辑、导出共用同一套组织隔离规则
- 网关、配置中心、数据库、服务部署一起联调
- 线上出现 404、504、空数据、串组织数据

## 核心模式

### 组织隔离

- 保存时落 `projectId`、`organizationId` 或等价字段
- 查询时优先按显式组织字段过滤
- 历史数据兼容逻辑单独标注，不要和主逻辑混在一起

### 配置排障顺序

1. 网关前缀是否正确
2. 服务是否部署到目标环境
3. 运行时读的是哪份配置
4. 数据库连的是哪套库
5. 表、索引、SQL 是否正确

### 接口联调

- 不只验证 HTTP 200
- 必看请求头里的组织字段
- 必看响应数据是否属于当前组织

### 前后端协同契约

- 列表、详情、编辑、导出接口的字段语义保持一致
- 动态标题、状态文案、名称文案如果后端已返回展示值，不要再要求前端本地映射修补
- 权限、组织范围、资源编码的语义在不同接口之间要对齐
- 如果某个问题本质上属于公共层缺失，不要把它长期变成单个业务接口的补丁

## 快速参考

- 请求头组织字段错：优先看前端上下文
- 请求头对但数据错：优先看保存字段或查询过滤
- 线上 504：优先看网关、配置、数据库，不要先猜代码

## 常见错误

- 只保存叶子节点 ID，不保存组织范围字段
- 只在前端筛选，后端不做组织隔离
- 配置中心切了，但应用实际没读到新配置
- 列表和详情返回的字段语义不一致
- 后端返回了动态展示文案，前端还得二次翻译或本地修正文案
