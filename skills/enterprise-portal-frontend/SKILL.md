# 企业门户前端

## 概述

这个技能用于处理企业门户前端本身的导航、应用入口、组织上下文传递和微应用承载问题。

开始处理门户壳页、公共 UI、多语言、宿主路由同步前，先读 `bluexii-components2` 技能。

核心原则：

1. 门户页是上下文源头，不只是跳板。
2. 当前组织必须稳定传递给应用入口页和微应用。
3. 壳页展示、网络请求、微应用收到的上下文必须一致。

## 何时使用

- 企业门户首页、工作台、应用中心开发
- 应用打开方式为同页直出、iframe、微前端 SDK
- 切组织后子应用数据不刷新

## 核心模式

### 固定接入顺序

1. 先确认门户是宿主、直出页面还是子应用承载层
2. 再确认应用入口是同源、跨域 `iframe` 还是模块式 React 子应用
3. 再确认 runtime 要下发哪些上下文
4. 最后才处理页面 UI 和入口行为

### 上下文传递

- 同源场景可读壳存储
- 跨域场景靠 runtime / SDK / `postMessage`
- 门户变更组织后，要触发子应用同步

### 宿主 runtime 接管

- 宿主优先统一下发 token、locale、resources、path
- 多语言控制权由宿主决定，不给业务仓留第二套入口
- 本地调试语言切换入口只能复用公共实现

### 子应用路由同步

- 宿主首次打开时必须把内部 `path` 回灌给子应用
- 子应用切页后必须上报最新 `path`
- 宿主外层 URL、标签缓存、刷新恢复要同步更新
- 同一路径同步要做幂等去重，避免 `navigate`、`replace`、`postMessage` 闭环
- 菜单选中态由当前路由推导，不做反向纠偏

### 应用入口验证

- 打开的 URL 正确
- 当前组织正确
- 应用标题、菜单、选中态正确
- 子应用刷新后能回到真实内部页

## 常见错误

- 门户显示组织 A，但子应用请求仍是组织 B
- 只改 portal UI，不改子应用上下文桥
- 不区分同源直出和跨域 iframe
- 外层 URL 不变，但子应用内部自己在切页
- 宿主接管语言后，子应用仍保留本地切换入口
