# PC 前端

## 概述

这个技能用于处理壳平台里的 PC 前端页面接入、菜单路由、树表联动和组织切换问题。

开始处理公共壳层、公共组件、公共分页、公共表单规范前，先读 `bluexii-components2` 技能。

核心原则：

1. 菜单、路由、资源编码必须一一对应。
2. 树、筛选、列表、详情、新增、编辑必须共用一套业务模型。
3. 切组织后必须重新拉取关键数据。

## 何时使用

- 接入基座 PC 页面
- 左侧树、顶部菜单、右侧列表一起联调
- 切组织后出现旧数据残留
- 页面显示正常但菜单选中态异常

## 核心模式

### 固定接入顺序

1. 先确认当前页属于列表页、详情页、表单页、双栏页、统计页还是弹窗/抽屉页
2. 再确认哪些能力必须直接复用公共仓
3. 再确认业务差异是公共配置项还是业务仓定制
4. 最后才写本地页面代码

### 页面一致性

- 树接口来源统一
- 列表筛选字段统一
- 新增编辑回填字段统一
- 列表、详情、表单不要各写一套近似页面骨架

### 公共骨架优先

- 列表页优先对标 `PcListPageLayout`
- 工具栏优先对标 `PcToolbar`
- 表格底部分页优先对标 `PcTableBottomBar`
- 表单录入优先对标 `FormEntrySpec`
- 不要在业务仓平行重写壳层、筛选区、分页区、必填标识方案

### 组织切换

- 优先读宿主上下文
- 页面缓存重时，用组织 ID 作为重挂 key
- 不能只依赖 token

### 宿主与多语言接管

- 多语言统一走 `@bluexii/shared-i18n`
- `DEV` 本地语言切换只复用公共入口
- 宿主传入 `locale/language` 后，本地切换入口必须隐藏或失效
- `QA / UAT / Prod` 下本地调试切换入口不可见

### 路由与菜单同步

- 路由决定菜单，不做菜单反向纠偏路由
- 同一路径的 `navigate`、`replace`、`postMessage` 要做幂等去重
- 跨域 `iframe` 或模块式 React 子应用必须验证宿主 URL、标签缓存、内部路由一致

### 浏览器验证

- 看壳页当前组织
- 看请求头 `x-organization-id`
- 看返回数据是否属于当前组织
- 看分页是否真的驱动请求
- 看 `dist/index.html` 是否残留根路径 `/assets/...`

## 常见错误

- 只修列表，不修新增编辑
- 菜单 key 和资源 code 不一致
- 组件切页后重新初始化，导致选中态丢失
- 业务仓自己写一套 `DevLocaleSwitcher`
- 页面布局看起来差不多，但没有复用公共骨架
- 宿主接管语言后，本地切换入口仍可操作
