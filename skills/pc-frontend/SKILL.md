# PC 前端

## 概述

这个技能用于处理壳平台里的 PC 前端页面接入、菜单路由、树表联动和组织切换问题。

核心原则：

1. 菜单、路由、资源编码必须一一对应。
2. 树、筛选、列表、详情、新增、编辑必须共用一套业务模型。
3. 切组织后必须重新拉取关键数据。

## 何时使用

- 接入基座 PC 页面
- 左侧树、顶部菜单、右侧列表一起联调
- 切组织后出现旧数据残留
- 页面显示正常但菜单选中态异常

## 核心模式

### 页面一致性

- 树接口来源统一
- 列表筛选字段统一
- 新增编辑回填字段统一

### 组织切换

- 优先读宿主上下文
- 页面缓存重时，用组织 ID 作为重挂 key
- 不能只依赖 token

### 浏览器验证

- 看壳页当前组织
- 看请求头 `x-organization-id`
- 看返回数据是否属于当前组织

## 常见错误

- 只修列表，不修新增编辑
- 菜单 key 和资源 code 不一致
- 组件切页后重新初始化，导致选中态丢失
