# 企业门户后端

## 概述

这个技能用于处理企业门户后端服务，包括应用详情、资源树、权限、组织下的应用可见性和应用入口元数据。

核心原则：

1. 门户后端返回的是应用运行契约，不只是展示信息。
2. 应用详情、资源树、权限校验必须按组织一致。
3. 错的入口元数据会直接导致前端走错容器或错上下文。

## 何时使用

- 开发或联调应用列表、应用详情、资源树、agent 检查
- 不同组织看到的应用不一致
- 子应用入口 URL、资源树、权限异常

## 核心模式

### 门户契约

- 应用详情中的 URL、agentId、资源信息正确
- 资源树与前端菜单语义一致
- 权限和组织范围一致

### 联调关注点

- `application/detail`
- `application/resource/tree`
- `application/agent/check`
- 当前组织下的可见应用列表

## 常见错误

- 应用详情属于当前组织，但子应用实际入口不适合当前容器
- 资源树 code 和前端路由不一致
- 权限返回正常，但组织范围错导致看到不该看的应用
