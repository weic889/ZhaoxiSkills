# 基座应用微服务集成验证清单

这份清单用于配合 `platform-microservice-integration-playbook` 使用，适合在每次联调、提测、上线验证时逐项勾选。

## 1. 接入建模

- [ ] 明确应用运行方式：壳页直出 / 跨域 iframe / 微前端 SDK
- [ ] 明确当前组织真值来源
- [ ] 明确左树或导航树接口是否来自平台统一服务
- [ ] 明确列表、新增、编辑共用的组织与业务字段模型

## 2. 前端上下文

- [ ] `x-organization-id` 不是直接优先取 token
- [ ] 跨域 iframe 场景已接入宿主 runtime 或 `postMessage`
- [ ] 宿主组织切换后，页面能重新拉树和列表
- [ ] 菜单选中态在二级页面跳转后不丢失

## 3. 后端保存

- [ ] 新增接口会落组织范围字段
- [ ] 新增接口会落关键业务主维度字段
- [ ] 编辑接口不会把组织字段改空或改错
- [ ] 保存链路能支撑后续列表和详情回填

## 4. 后端查询

- [ ] 列表按组织范围字段过滤
- [ ] 历史数据兼容逻辑有明确边界
- [ ] 树相关查询不会绕过组织过滤

## 5. 配置与部署

- [ ] 线上网关路径正确
- [ ] 服务读取的是目标环境配置
- [ ] 数据库连接的是目标环境库
- [ ] 必要表已存在
- [ ] CI / 镜像 / webhook 已完成

## 6. 浏览器线上验证

- [ ] 壳页展示的组织名称正确
- [ ] 树接口请求参数中的组织正确
- [ ] 列表接口请求头中的 `x-organization-id` 正确
- [ ] 列表返回的数据属于当前组织
- [ ] 新增数据后刷新页面仍属于当前组织
- [ ] 切换到另一个组织后旧组织数据不残留

## 7. 故障定位判断

如果壳页组织正确，但请求头组织错误：

- 结论：前端上下文传递有问题

如果请求头组织正确，但返回数据错误：

- 结论：后端保存字段或查询过滤有问题

如果树切了组织，列表没切：

- 结论：页面缓存、依赖更新或组件重挂有问题

如果接口路径 404 / 504：

- 结论：先看网关前缀、部署状态、配置和数据库
